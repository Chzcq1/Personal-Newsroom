========================================
SPRINT 9
CONTEXTUAL INTELLIGENCE LAYER

Mission:

INFOX must evolve from keyword-based personalization into contextual intelligence.

Current problem:

Users still receive irrelevant stories because the system matches keywords instead of understanding semantic relevance and narrative context.

Goal:

The feed should feel:

“INFOX understands what I actually care about.”

NOT:

“It found a matching word.”

========================================
TASK A — INTEREST GRAPH ENGINE

Current system:

Interest = simple keyword list.

Upgrade to:

Interest Graph system.

Example:

BTC
→ Bitcoin ETF
→ BlackRock
→ Coinbase
→ SEC
→ Mining
→ Stablecoins
→ Ethereum ecosystem

Requirements:

* create interestGraph.ts
* define related entities
* support weighted relationships
* allow future dynamic expansion
* graph should influence feed ranking

Users interested in BTC should receive:

high-value Bitcoin ecosystem developments

NOT generic “crypto mentions.”

========================================
TASK B — SEMANTIC RELEVANCE CLASSIFIER

Current issue:

Keyword matching causes noisy feeds.

Implement semantic relevance scoring.

Every article should be classified as:

* Directly relevant
* Contextually relevant
* Weakly related
* Incidental mention

Example:

“Bitcoin ETF inflows surge”

→ Direct relevance

“Startup funded by crypto investors”

→ Weak relevance

Requirements:

* create relevanceClassifier.ts
* combine:
    keyword score
    entity overlap
    topic similarity
    narrative relevance
* expose relevance score in debug mode

Feed ranking must prioritize:

direct relevance first.

========================================
TASK C — NARRATIVE CLUSTERING ENGINE

Current issue:

Multiple articles repeat the same narrative.

Implement narrative clustering.

Examples:

10 ETF articles
→ grouped into:
“Bitcoin ETF momentum accelerating”

Requirements:

* cluster similar stories
* generate narrative headlines
* merge duplicate coverage
* preserve source diversity

Feed should display:

Narrative
↓
Supporting sources
↓
Why it matters

instead of endless repeated headlines.

========================================
TASK D — USER TASTE LEARNING

Current issue:

Feed personalization is static.

Implement adaptive taste learning.

Track:

* opened articles
* saved briefings
* ignored topics
* reading completion
* recurring engagement patterns

Requirements:

* local-only for now
* transparent logic
* no dark patterns
* no addictive optimization

The system should gradually learn:

what the user truly values.

========================================

Goal:

Understand WHY bad recommendations happen.
