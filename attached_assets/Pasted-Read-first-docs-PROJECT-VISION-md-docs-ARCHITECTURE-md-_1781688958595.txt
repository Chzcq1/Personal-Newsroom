========================================
SPRINT 20
SYSTEM CONSOLIDATION & PRODUCTION PREPARATION
========================================

MISSION

INFOX has reached the point where feature velocity is creating architectural fragmentation.

The system now contains:

* overlapping dashboards
* duplicated preview systems
* scattered configuration logic
* fragmented Telegram UX
* excessive admin tooling
* growing documentation overload

This sprint exists to:

STOP SYSTEM SPRAWL

and prepare INFOX for:

* real deployment
* real users
* authentication
* payments
* production runtime separation

This is a cleanup + stabilization sprint.

NOT a feature explosion sprint.

========================================
PRIMARY OBJECTIVES
========================================

1. Consolidate duplicated systems

2. Reduce UI complexity

3. Separate user/admin architecture

4. Prepare production runtime outside Replit

5. Simplify operational visibility

6. Prepare clean foundation for Sprint 21 auth

========================================
NON-NEGOTIABLE RULES
========================================

1. ONE SYSTEM PER RESPONSIBILITY

Only prepare architecture.
