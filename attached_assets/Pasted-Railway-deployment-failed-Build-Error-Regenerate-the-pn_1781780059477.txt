## Problem
The build fails at the `pnpm install --frozen-lockfile --prefer-offline` step with `ERR_PNPM_LOCKFILE_CONFIG_MISMATCH`. The `overrides` configuration in `pnpm-workspace.yaml` does not match what is recorded in `pnpm-lock.yaml`.

Build log error:
```
ERR_PNPM_LOCKFILE_CONFIG_MISMATCH  Cannot proceed with the frozen installation. The current "overrides" configuration doesn't match the value found in the lockfile
Update your lockfile using "pnpm install --no-frozen-lockfile"
```

The service uses the RAILPACK builder with pnpm v9.15.9 in a monorepo with 9 workspace projects. The build stage failed; no railway.json or railway.toml config file is in use.

## Diagnosis
`pnpm-workspace.yaml` defines extensive `overrides` that exclude non-linux-x64 platform binaries for esbuild, lightningcss, @tailwindcss/oxide, rollup, and @expo/ngrok-bin, and also pins esbuild to 0.27.3 and replaces @esbuild-kit/esm-loader with tsx. The `pnpm-lock.yaml` was generated when these overrides were different or absent, so the frozen install correctly rejects the mismatch. The triggering commit only changed UI translation files and added a ThemeContext — no changes were made to `pnpm-workspace.yaml` or `pnpm-lock.yaml`, confirming the mismatch predates this commit. This is the first deployment of this service.

## Fix
1. **File:** `pnpm-lock.yaml` (repo root)
2. **Action:** Regenerate by running locally:
   ```bash
   pnpm install
   ```
   This updates `pnpm-lock.yaml` to reflect the current `overrides` in `pnpm-workspace.yaml`.
3. **Commit** the updated `pnpm-lock.yaml` and push to `main`.
4. This resolves the mismatch because the lockfile will now contain the same overrides configuration that pnpm reads from `pnpm-workspace.yaml` at install time.
