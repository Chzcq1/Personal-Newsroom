========================================
TASK A — SHARED BRIEFING CACHE

Current problem:

Every user request generates a fresh AI briefing.

This burns token budget repeatedly.

Implement:

Global Briefing Cache

Requirements:

* Same topic + same hour = same briefing
* Store generated briefing
* Reuse briefing for all users

Example:

Technology briefing generated at 09:00

User A requests 09:05
→ use cached briefing

User B requests 09:20
→ use cached briefing

User C requests 09:55
→ use cached briefing

Only regenerate after cache expiry.

Cache duration:

60 minutes

Add:

cache hit rate metrics

========================================
TASK B — ARTICLE PREPROCESSOR

Current problem:

Too much article text sent to AI.

Implement preprocessing layer.

Before AI call:

* remove duplicate articles
* remove boilerplate text
* trim descriptions
* extract only useful content

Target:

Reduce payload size by 60–80%.

Log:

before tokens
after tokens

TASK C — TOKEN BUDGET CONTROLLER

Current problem:

Technology and AI topics exceed GitHub limits.

Implement strict limits.

Maximum articles:

5

Maximum article length:

1000 characters

Maximum total prompt size:

6000 tokens

If limit exceeded:

drop lowest ranked articles.

Never exceed provider limit.

Goal:

Higher quality briefing.
